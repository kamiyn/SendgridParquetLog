- 調査: 現行 Home.razor の UI ロジックと DuckDB 連携の残骸コードを確認
- 調査: 参照プロジェクト BlazorDuck の DuckDB Wasm 連携実装 (DuckDbQueryService と duckdbInterop.js) を確認
- 方針検討: DuckDB Wasm をブラウザ側で初期化し、月次マニフェスト取得と日別パーケット読込を JS 側で行う構成を設計
- 実装: DuckDB WASM 設定用オプション `DuckDbWasmOptions` を追加し、既定値を `/lib/duckdb-wasm` 配下のアセットに設定
- 実装: `Program.cs` で `DuckDbWasmOptions` を構成にバインドし、起動時バリデーションを追加
- 設定: `appsettings.json` に DuckDB WASM の既定パス設定を追加
- 実装: `wwwroot/js/duckdbInterop.js` を追加し、DuckDB WASM の初期化・月マニフェスト取得・日別 Parquet 読込・検索クエリ実行・破棄処理を実装
- 実装: Home.razor で DuckDB オプションを注入し、JS モジュール初期化・月マニフェスト保持・検索クエリ発行のフローを刷新

- 後処理: DuckDB JS モジュール破棄時に `dispose` 呼び出しと `IJSObjectReference.DisposeAsync` の両方を実行するよう調整
- 検証: `dotnet build` を実行し、DuckDB WASM アセットのダウンロードとビルド成功を確認
- フォーマット: プロジェクト規約に従い `dotnet format` を実行
- 設定: `SendgridParquetViewer/wwwroot/lib/duckdb-wasm/` を `.gitignore` に追加し、ダウンロード済み WASM アセットをコミット対象から除外
- 実装: DuckDB Wasm バンドル生成用の `DuckDbBundle/` (Vite + npm プロジェクト) を追加し、出力先を `SendgridParquetViewer/wwwroot/lib/duckdb-wasm` に設定
- 設定: DuckDB Module Loader の既定値を `duckdb-browser-bundle.js` に変更し、アプリ設定とブラウザ JS 側の既定も合わせて更新

- リファクタ: DuckDB インターロップ実装を `DuckDbBundle/src/index.js` に統合し、AsyncDuckDB 初期化・月データ管理・クエリ実行・破棄処理をモジュール内で提供
- リファクタ: Home.razor から外部 JS を読み込む際に DuckDB バンドルのモジュールパスを動的計算するヘルパーを追加し、バンドル経由で初期化するよう変更
