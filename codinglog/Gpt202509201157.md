# プロンプト
DuckDbService の GetEventsByDateAsync() が検索に失敗するため、クライアント側で duckdb-wasm を利用し、ブラウザに年月単位で Parquet を事前ダウンロードする仕組みを構築し、ダウンロード済み日付をカレンダー表示で選択できるようにする。また、ブラウザ内キャッシュをハッシュで検証して再利用し、S3 には compaction 時に SHA256 メタデータを書き込むこと。

# 処理内容
- S3 アップロードでカスタムメタデータを付与できるよう `S3StorageService` を拡張し、SHA256 メタデータ取得 API を追加。
- `CompactionService` で Parquet を生成する際に SHA256 ハッシュを計算し、S3 オブジェクトのメタデータとして保存。
- 月次の Parquet ファイル一覧とハッシュを提供する `ParquetCatalogService` と `ParquetController` を新設し、ブラウザから manifest とファイルを取得可能にした。
- `duckdb-wasm` を扱う `wwwroot/js/duckdbInterop.js` を追加。IndexedDB を用いた月次ダウンロード・ハッシュ検証・ファイル登録・クエリ実行を実装。
- `Home.razor` を全面刷新し、月選択とカレンダー UI、duckdb-wasm 経由の検索、結果表示に対応。関連スタイルを `app.css` に追加。
- 不要になった `DuckDbService` を削除し、依存登録を整理。
- 新しい処理を `dotnet build` で検証。
- 追加の指示に従い、配列と `IReadOnlyList<T>` を活用するようビューワ実装とカタログサービスを調整し、可変 `List<T>` の露出を避けた。
