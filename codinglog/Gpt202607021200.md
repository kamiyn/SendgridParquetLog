# Compaction のメモリ使用量の抑制

## プロンプト

Documents/Improve202607.md Compaction のメモリ使用量の抑制 を実装してください

## 処理内容

Improve202607.md セクション 1（Compaction のメモリ使用量の抑制）の P0 / P1 タスクを実装。

### P0-1. RowGroup を概算バイト数で区切る

- `ParquetService.ConvertToParquetStreamingAsync` に `rowGroupMaxEstimatedBytes` パラメータを追加（既定 48 MiB）
- `IColumnBuffer` / `ColumnBuffer<T>` に `EstimatedBytes` を追加し、列ごとに概算バイト数を加算
- フラッシュ条件: 概算バイト数 >= しきい値 **または** 行数 >= `rowGroupSize`（安全上限）

### P0-2. RowGroupSize を設定化

- `CompactionOptions.RowGroupSize` を追加（既定 25,000）
- `CompactionService` / `MoreCompactionService` から定数 60,000 を削除し Options 参照へ

### P0-3. MaxBatchSizeBytes 既定見直し

- 既定値を 512 MiB → 256 MiB に変更（deploy.yml の運用値と整合）

### P1-4. ToArray() コピー削減

- `ColumnBuffer<T>` を `List<T>` から固定容量 `T[]` へ変更
- 満杯フラッシュ時は配列の所有権を `DataColumn` に移譲しコピー回避
- 部分フラッシュ時は `GC.AllocateUninitializedArray` + `Span.CopyTo` で 1 回コピーのみ
- `Clear()` で参照型列の使用済みスロットを `Array.Clear` し、部分フラッシュ時の二重保持を防止

### P1-5. GC / ランタイム設定

- **未実施**（ユーザー指示によりスキップ）

### P1-6. メモリ観測性

- `ParquetRowGroupFlushMetrics` 構造体と `onRowGroupFlushed` コールバックを追加
- `CompactionService` で `ZLogDebug` により行数・概算バイト・GC/WorkingSet を記録
